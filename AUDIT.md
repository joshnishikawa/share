# Codebase Audit

## 1) Security Vulnerabilities, Bugs & Broken Things
- [x] `script.js` `getCookieObject()`: Cookie keys after the first have a leading space (missing `.trim()`), so lookups by key name silently fail for all but the first cookie. **FIXED: Added .trim() to key.**
- [x] `study_activities.js` `chooseL2Sentence()`: `sentence.length > 1` check is copy-pasted from `chooseL2`'s word/letter branching. Sentences are always > 1 char, so `rightLetter`/`wrongLetter` branches are dead code. **FIXED: Removed dead branches, always uses rightSentence/wrongSentence.**
- [x] `study_activities.js` `typeL2()`: `sentence.match(new RegExp(word, 'i'))` will throw TypeError if `word` is not found in `sentence` (`.match()` returns null, then `null[0]` throws). **FIXED: Added null guard.**
- [x] `study_activities.js` `typeL2()`: `$("#backspace").on('click').on('click', fn)` â€” double `.on('click')` is a no-op first call, then attaches. Works but looks like a bug. **FIXED: Single .on('click').**
- [x] `study_activities.js` `typePart()`: Same double `.on('click')` as typeL2. **FIXED: Single .on('click').**
- [x] `study_activities.js` `typePart()`: Digraph splitter checks `letters[0]` and `letters[3]` â€” different logic from typeL2's profanity guard. **NOT A BUG: Intentionally different purpose (break visible digraphs vs prevent profanity). Comments updated to clarify.**
- [x] `study_activities.js` `typeSentence()`: `callback` is only called on failure, never on success. All other activities call `callback(true)` on success. **FIXED: Added callback(true) on success.**
- [x] `study_activities.js`: Promise `reject()` is never called in any function. Promises can hang forever if the activity hits an unexpected state. **ACCEPTED: Activities resolve on user interaction â€” no timeout needed. A hung promise just means the user left the page.**
- [x] `study_utilities.js` `playaudio()`: If `.play()` fails (browser autoplay policy / no src), promise hangs forever â€” no `.catch()` on play() and no rejection. **FIXED: Added .catch() that resolves gracefully.**
- [x] `study_utilities.js` `getRandomSet()`: References a global `set` variable that must exist elsewhere â€” will throw ReferenceError if undefined. **NOT A BUG: This file is shared across projects. `set` is defined by the consuming page. Comments updated.**
- [x] `study_utilities.js` `next()`: Keyboard path hides `#next` but does NOT restore `#instructions` visibility (click path does) â€” keyboard users lose the instructions. **FIXED: Unified done() handler for both paths.**
- [x] `study_utilities.js` `next()`: Neither handler removes the other â€” both can fire, resolving the promise twice. **FIXED: done() removes both handlers before resolving.**
- [x] `study_utilities.js` `randIndexOf()`: `indexes` has no `let`/`const`/`var` â€” leaks as a global variable. **FIXED: Added `let`.**
- [x] `study_utilities.js` `progressPie`: `querySelector` runs at parse time â€” if DOM doesn't have `.left-half`/`.right-half` yet, properties are null and `.progress()` will throw. **FIXED: Lazy-init on first call.**
- [x] `study_utilities.js` `getGapFill()`: `RegExp.esc` is defined but never actually used on the word `i` â€” if a word contains regex metacharacters, the match breaks. **FIXED: Now uses RegExp.esc(i) in the RegExp constructor.**
- [x] `auth_widget.js` `fetchUser()`: No `response.ok` check â€” a 500 response will try to parse JSON and throw a misleading error. **FIXED: Added response.ok guard.**
- [x] `sync_manager.js` `mergeData()`: Calls `window.location.reload()` then continues to call `syncToServer()` â€” the sync may not complete before the page reloads (race condition). Should sync first, then reload. **FIXED: syncToServer() now fires before reload().**
- [x] `sync_manager.js` `syncToServer()`: The debounced callback is async inside setTimeout, so the returned promise from `syncToServer()` resolves immediately (before the actual sync). Callers can't await the real sync completion. **ACCEPTED: Fire-and-forget by design. The mergeData() reload path was already fixed to call syncToServer() before reload.**
- [x] `srs.js`: File is empty (0 lines) â€” either dead code or a placeholder. Remove or implement. **ACCEPTED: May be used by other projects sharing this codebase. Left as-is.**
- [x] `progress.js`: `progressPie` querySelector runs at parse time â€” null if DOM not ready (same bug as study_utilities.js). **FIXED: Lazy-init on first call.**
- [x] `canvas_builder.js` `loadFromStorage()`: `JSON.parse()` with no try/catch â€” corrupted localStorage will throw and break the canvas. (`sync_manager.js` handles this defensively; this doesn't.) **FIXED: Added try/catch, resets on corruption.**
- [x] `canvas_builder.js` `generateThumbnail()`: Returns `''` immediately; thumbnail arrives asynchronously. First save after adding an item won't include thumbnail. **FIXED: saveToStorage() is now async and awaits generateThumbnailAsync() before saving. Dead sync wrapper removed.**
- [x] `canvas_manager.js` `setupEventHandlers()`: Called again after clone without unbinding previous handlers â€” duplicate event listeners accumulate. **FIXED: Added .off() before .on() at top of function.**
- [x] `abc.js` `loadAnActivity()`: catch block marks the card as correct on any error â€” hides bugs by silently advancing. **FIXED: Now logs error and skips without marking correct.**
- [x] `abc.js`: Missing letter 'x' from the alphabet array (intentional? or oversight). **FIXED: Added 'x' with vocab [xylophone, x-ray, fox] and x_sound.**
- [x] `choose.js` `showSelectedImage()`: Builds a jQuery attribute selector by interpolating `word` unsanitized â€” `$('img[alt="' + word + '"]')`. If a word contains `"` or `]`, the selector breaks; if data is adversarial it could match unintended elements. Should use `.filter()` instead. **FIXED: Replaced with .filter().**
- [x] `choose.js`: `recognition.onresult` reads `event.results[event.results.length - 1][0].transcript` â€” only the first alternative (index 0) is checked despite `maxAlternatives = 3`. The extra alternatives are wasted. **FIXED: Now loops all alternatives.**
- [x] `universal.css`: All gradient color classes use ONLY `-webkit-linear-gradient` â€” missing the standard `linear-gradient()` fallback. Firefox and non-Webkit browsers see only the flat `background-color`. **FIXED: Added standard linear-gradient() fallback to all gradient classes.**
- [x] `universal.css` `@font-face`: Font paths use `/public/files/` â€” depends on Express mounting `public/` at `/public/` rather than the conventional `/`. Verify server config; may silently 404. **FIXED: Express mounts at `/`, paths corrected to `/files/`.**
- [x] `style.css` `.flip-back`: Two CSS declarations crammed onto one line (`align-items: center;  position: absolute;`) â€” easy to miss during editing. **FIXED: Split to separate lines. Also removed unnecessary vendor-prefixed backface-visibility from both .flip-back and .flip-front.**
- [x] `head.ejs`: jQuery UI CSS (`jquery-ui.css`) is loaded but the jQuery UI JS library is never loaded â€” the CSS is dead weight unless a specific page loads jQuery UI JS separately. **FIXED: Removed.**
- [x] `head.ejs`: No Content Security Policy (`<meta http-equiv="Content-Security-Policy">`) â€” all inline `<script>` blocks execute freely. Not a bug, but no XSS mitigation layer. **DEFERRED: Would require nonce-based CSP across all views. Track separately.**
- [x] `head.ejs`: Bootstrap 5.1.3 is ~2 major versions behind current (5.3.x). **DEFERRED: Upgrade when ready â€” no breaking security issue in 5.1.3.**
- [x] `error.ejs`: No `<html>`, `<head>`, or `<body>` wrapper â€” renders a bare HTML fragment. No head.ejs include. **FIXED: Now includes head.ejs and foot.ejs like all other views.**
- [x] `404.ejs`: `start` and `submitAnswer` are implicit globals (arrow functions not assigned to `let`/`const`). `$(window).on('keydown')` is never unbound after game completion. **FIXED: Added const, unbind keydown+keyup on correct answer.**
- [x] `dictionary.ejs` **XSS**: `showWord()` inserts `w.word`, `w.meaning`, and `w.image` directly into HTML via template literal â€” no escaping. A malicious translation or vocab entry could inject scripts. **FIXED: Added escapeHtml().**
- [x] `dictionary.ejs`: `processWord()` result is destructured to `processedEnglish`/`processedJapanese` but these variables are NEVER USED â€” the original `EnglishWord`/`JapaneseWord` are passed to `showWord()` instead. Dead code. **FIXED: Now uses processWord result.**
- [x] `dictionary.ejs`: `new SpeechRecognition()` runs unconditionally but the speech button is commented out in the HTML â€” will throw in browsers without Web Speech API. **FIXED: Wrapped in if (SpeechRecognition) guard.**
- [x] `dictionary.ejs`: `recognition.onend` clears `#output` even after a successful result â€” the recognized word may flash and disappear. **FIXED: Removed the onend handler.**
- [x] `multiplayer.ejs` **XSS**: `updatePlayerList()` inserts `p.id` and `p.color` from socket data directly into HTML template literals. A malicious player name could inject HTML/JS into all other players' browsers. **FIXED: Added escapeHtml() + sanitizeColor().**
- [x] `multiplayer.ejs` **XSS**: `roomSearch` handler inserts `p.id` and `p.color` via string concatenation â€” same injection risk as above. **FIXED: Same escapeHtml() + sanitizeColor().**
- [x] `navbar.ejs`: Uses `data-toggle="tab"` (Bootstrap 4 attribute) â€” should be `data-bs-toggle="tab"` for Bootstrap 5. Tabs likely don't auto-toggle via Bootstrap JS. **FIXED: Updated to data-bs-toggle.**

## 2) Inconsistencies in Language, Approach & Structure
- [x] `script.js`: All functions are implicit globals (no `const`/`let`/`var` declaration). Intentional for cross-file access, but fragile â€” any typo creates a new global silently. **NOTED: Intentional pattern for shared cross-file access. Modernizing would require a module bundler.**
- [x] `script.js`: Mix of `var` and `let` within the same file (`FYshuffle` uses both, `parallelShuffle` uses only `var`). **FIXED: All `var` â†’ `let` in both functions.**
- [x] `script.js`: `FYshuffle` returns the array; `parallelShuffle` does not. Callers have to know which is which. **FIXED: `parallelShuffle` now returns `[a, b]`.**
- [x] `study_activities.js`: Event cleanup is inconsistent â€” some functions use `$(document).off('keydown')`, others use `document.onkeydown = function(){}`, some do neither. Two fundamentally different approaches to the same problem. **FIXED: All standardized to jQuery `.on('keydown')` / `.off('keydown')`.**
- [x] `study_activities.js`: Callback signatures differ â€” most activities call `callback(true/false)`, but `sortPics` and `sortWords` call `callback({success: bool, word: string})`. **NOTED: The object form is needed for SRS tracking â€” `word` identifies which item was answered. Different signatures serve different needs.**
- [x] `study_activities.js`: Wrong-answer CSS class is inconsistent â€” some use `bg-primary` â†’ `bg-danger`, others use `alert-primary` â†’ `bg-danger`. **NOT A BUG: `alert-primary` is intentional for pic cards (lighter background to show images).**
- [x] `study_activities.js`: `gapFill` has no keyboard support, while nearly identical `partFill` does. **FIXED: Added keyboard handler for single-word gapFill, matching partFill pattern.**
- [x] `study_utilities.js`: HTML builders use implicit globals (arrow functions without `const`/`let`) while teach/display functions use `function` declarations â€” mixed declaration styles. **NOTED: Implicit globals are the shared-file pattern (see script.js). Cosmetic inconsistency only.**
- [x] `study_utilities.js`: `rightPic`/`wrongPic` use `alert-primary` while `rightWord`/`wrongWord` use `bg-primary` â€” CSS class inconsistency (also affects study_activities.js wrong-answer styling). **NOT A BUG: Intentional â€” pics need lighter background for image visibility.**
- [x] `study_utilities.js` `getRandomSet()`: Callback parameter `data` shadows the outer parameter also named `data`. **NOTED: Comments updated.**
- [x] `study_utilities.js` `flashCard()`: Uses `await` on jQuery `.html().show()` which aren't real Promises â€” works because `.show()` is synchronous, but `await` is misleading. **FIXED: Removed misleading `await` on jQuery `.html().show()`.**
- [x] `study_utilities.js` `clearTable()`: Wraps synchronous jQuery ops in `Promise.all()` â€” unnecessary complexity. **FIXED: Simplified to plain synchronous function.**
- [x] `study_utilities.js`: `RegExp.esc` is monkey-patched onto `RegExp` constructor identically in both `getGapFill` and `getPartFill`. **FIXED: Hoisted to single definition above both functions.**
- [x] `auth_widget.js`: Uses ES6 class, `fetch`, `async/await`, `const` â€” a different era of JS than all other frontend files which use jQuery, `var`/`let` globals, and Promises. The quality gap highlights how much the older files could benefit from modernization. **NOTED: Observation â€” represents the target style for future modernization.**
- [x] `auth_widget.js`: The ONLY file that escapes HTML before inserting user data. All `study_utilities.js` template builders insert values raw. **NOTED: study_utilities.js data comes from server-controlled vocabulary, not user input. XSS risk is low but worth revisiting if user-generated content is ever displayed.**
- [x] `progress.js` vs `study_utilities.js`: `progressToPercent`, `progressPie`, and `progressPie.slammer` are defined in BOTH files with slightly different signatures (`progress.js` has no `base` param). Risk of divergence. **NOTED: Different formulas for different contexts (letters vs SRS). Loaded on separate pages (letters.ejs / SRS.ejs) so no conflict. `progressPie`/`progressPie.progress` are identical â€” could share, but low priority.**
- [x] `canvas_builder.js` / `canvas_manager.js`: Use constructor functions (not ES6 classes like `auth_widget.js` / `sync_manager.js`). Functional style is fine but inconsistent with the newer files. **NOTED: Cosmetic. Constructor functions work correctly â€” refactoring to class syntax offers no functional benefit.**
- [x] `canvas_builder.js`: Stores full innerHTML + base64 thumbnails in localStorage â€” can grow very large and hit browser limits (typically 5-10MB). **ACCEPTED: Known limitation. Mitigation would require IndexedDB or server-side storage â€” out of scope.**
- [x] `dressup.js`: Uses `var` throughout while newer files use `let`/`const`. Also has many `console.log` statements left from debugging. **FIXED: All `var` â†’ `let`/`const`, all `console.log` removed.**
- [x] `abc.js`: Uses `const` for data, `async`/`await` for flow â€” more modern than study_activities/utilities but still relies on implicit globals (`aim`, `update`, `checkProgress`). **NOTED: Same implicit-global pattern as other shared files. Modern otherwise.**
- [x] `choose.js`: Well-structured IIFE with proper teardown, `let`/`const`, clean socket event management â€” closest quality to `auth_widget.js` among the older files. Good reference for the multiplayer pattern. **NOTED: Positive observation â€” good reference for future multiplayer activities.**
- [x] `match.js` / `race.js`: Stub files â€” only emit a ready signal. Server-side counterparts are also stubs. Listed in the multiplayer menu but have no actual gameplay. **NOTED: Placeholder for future development.**
- [x] `universal.css` / `style.css`: `.inner-shadow` is defined identically in BOTH files. Same for `.YM` (Yusei Magic font shortcut). Duplicates that will diverge. **FIXED: Removed duplicates from style.css, kept in universal.css.**
- [x] `universal.css` `.fc-clone`: Manually replicates Bootstrap's `.form-control` â€” will drift on Bootstrap upgrade. **ACCEPTED: Intentional clone for customized styling. Low risk since Bootstrap upgrades are infrequent.**
- [x] `universal.css`: All 18 text-color classes use `!important` â€” aggressive specificity that limits composability. **ACCEPTED: Needed to override Bootstrap's specificity. Removing `!important` would break color overrides in Bootstrap contexts.**
- [x] `style.css` `.flip-back` / `.flip-front`: 5 vendor-prefixed `backface-visibility` declarations including `-o-` (Opera never used this prefix). Overkill for modern browsers. **FIXED (session 2): Removed vendor prefixes.**
- [x] `customIcons.css`: Loads `.eot` (IE) and `.svg` (deprecated Safari) font formats â€” unnecessary if only targeting modern browsers. **FIXED: Stripped .eot and .svg formats, kept .woff and .ttf only.**
- [x] `head.ejs` vs individual views: Some views use `include('../head')` (no extension), others use `include('../head.ejs')` â€” both work in EJS but inconsistent across files. **FIXED: Standardized all includes to no `.ejs` extension + `-%>` whitespace trimming.**
- [x] `navbar.ejs`: Uses Bootstrap 4 `data-toggle` instead of Bootstrap 5 `data-bs-toggle` â€” the tabs are navigated via full page loads (`href="letters"` etc.) not JS toggling, so it works, but the attribute is misleading. **FIXED (session 2): Updated to `data-bs-toggle`.**
- [x] `multiplayer.ejs`: Uses `var` throughout its 570-line `<script>` block while the client-side `choose.js` it loads uses `let`/`const`. Different eras of code within the same feature. **FIXED: All 33 `var` â†’ `let`/`const` (const for never-reassigned, let for mutables).**
- [x] `error.ejs`: Renders bare HTML fragment with no `<html>/<body>` wrapper, unlike every other view that includes head.ejs + foot.ejs. **FIXED (session 2): Now includes head.ejs + foot.ejs.**
- [x] `dictionary.ejs`: Has Speech Recognition JS fully wired up but the HTML button is commented out â€” split state between markup and script. **FIXED: JS block commented out to match HTML. Both can be uncommented together to re-enable.**
